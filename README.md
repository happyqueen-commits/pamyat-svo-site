# Книга памяти СВО — MVP архива

Проект на Next.js + Prisma + SQLite.

## Что есть в этой версии
- публичный архив карточек
- страница отправки материалов с масками и защитой от простого спама
- загрузка файлов к заявке
- личный кабинет автора
- единый вход для автора, модератора и администратора
- редакторская очередь для модерации материалов
- управление ролями пользователей
- базовый журнал аудита

## Быстрый запуск
```bash
npm install
npm run setup
npm run dev
```

Если проект уже запускался раньше, лучше пересоздать локальную базу:

### PowerShell
```powershell
Remove-Item prisma/dev.db -Force -ErrorAction SilentlyContinue
Remove-Item .next -Recurse -Force -ErrorAction SilentlyContinue
npm install
npm run setup
npm run dev
```

## Тестовые аккаунты после `npm run setup`
- администратор: `admin@memory.local` / `Admin12345!`
- модератор: `moderator@memory.local` / `Moderator123!`
- автор: `author@memory.local` / `Author123!`

## Основные маршруты
- `/` — главная
- `/archive` — публичный архив
- `/submit` — отправка материала
- `/auth` — вход и регистрация
- `/cabinet` — личный кабинет
- `/cabinet/moderation` — очередь модерации
- `/cabinet/users` — управление пользователями и ролями

## Что изменено по архитектуре
- публичная часть больше не содержит явной «админки»
- служебный доступ переведён на роли и cookie-сессии
- заявка автоматически привязывается к пользователю, если он авторизован
- действия модераторов и администраторов пишутся в аудит
- добавлены базовые ограничения частоты запросов для входа и отправки материалов


## Новое в версии v11.4

- автор может открыть свою заявку из кабинета и отредактировать её до публикации
- для отклонённых и черновых материалов доступна повторная отправка на проверку
- кабинет переработан в более рабочий dashboard-формат
